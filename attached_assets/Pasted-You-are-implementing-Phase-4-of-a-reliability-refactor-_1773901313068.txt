PART A — Schema Fixes (do this first, then migrate)

1. ADD composite index on campaign_sends(campaign_id, status)
   In shared/schema.ts, find the campaign_sends table indexes and add:
   index('campaign_sends_campaign_status_idx')
     .on(campaignSends.campaignId, campaignSends.status)
   Remove the now-redundant separate index on status alone 
   (keep the index on campaign_id alone — still useful for joins).

2. ADD functional index on lower(email)
   In shared/schema.ts, add to subscribers table:
   index('subscribers_email_lower_idx').on(sql`lower(email)`)
   This enables case-insensitive email lookups to use an index instead of seq scan.

3. ADD partial index on job queues for pending status
   For campaign_jobs and import_job_queue tables, add:
   index('campaign_jobs_pending_idx')
     .on(campaignJobs.status)
     .where(sql`status = 'pending'`)
   This makes job claiming queries fast even when most records are completed.

4. ADD cascading deletes on foreign keys
   In shared/schema.ts, update these foreign key references to include 
   .onDelete('cascade'):
   - campaign_sends.campaignId → campaigns.id
   - campaign_sends.subscriberId → subscribers.id  
   - campaign_stats.campaignId → campaigns.id
   - campaign_stats.subscriberId → subscribers.id
   - automation_enrollments.subscriberId → subscribers.id
   - automation_enrollments.automationId → automations.id
   
   IMPORTANT: After adding these, simplify the corresponding manual cascade 
   logic in storage.ts (the clearSubscriberDependencies method). You can remove 
   the manual DELETE loops for tables that now have DB-level cascades. 
   Keep the manual loop ONLY for tables without cascade (if any remain).

   Verify in the Neon console that the indexes and cascade rules exist.
